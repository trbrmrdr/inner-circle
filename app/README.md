# Ближний круг / Inner Circle

## Новая схема работы

`app/dst_0` - базовый слепок сайта. Его не редактируем руками: это reference-версия, из которой собирается рабочий сайт.

`app/src` - новые исходники. Сюда переносим повторяющиеся блоки, локальные стили, новые скрипты и будущие правки по страницам.

`app/dist` - собранный сайт для просмотра и хостинга. Папка создается командой сборки и не хранится в git.

## Команды

```bash
npm run build
npm run dev
npm run serve
```

- `npm run build` пересобирает `app/dist` из `app/dst_0` и `app/src`.
- `npm run dev` запускает локальный просмотр и пересобирает сайт при изменениях в `app/src`.
- `npm run serve` запускает просмотр без автоматической пересборки.

## Как устроена сборка

Сборщик копирует `app/dst_0` в `app/dist`, а затем подключает:

- `legacy.css` - бывший большой `index.css` из базового сайта;
- `legacy-runtime.js` - бывший большой `index.js` из базового сайта;
- `inner-circle.css` - собранные стили из `app/src/styles`;
- `inner-circle.js` - собранные скрипты из `app/src/scripts`.

`legacy.css` уже разложен по старым CSS-маркерам в `app/src/legacy-styles`. Это позволяет точечно работать с базовыми стилями компонентов, не открывая один большой файл. `legacy-runtime.js` пока остается цельным, потому что в нем держатся анимации, переходы и WebGL-поведение.

## Где править дальше

- `app/src/partials/headers` - общая шапка RU/EN.
- `app/src/partials/footers` - общие варианты футера.
- `app/src/partials/accordions` - вынесенные аккордеоны с картинками, размерами, ценами и описаниями.
- `app/src/legacy-styles` - базовые стили старого макета, разделенные по компонентам из экспортного CSS.
- `app/src/styles/components` - стили общих блоков.
- `app/src/styles/pages` - стили конкретных страниц.
- `app/src/styles/features` - отдельные функции, например карта контактов.
- `app/src/scripts` - новые JS-правки поверх legacy runtime.

## Правило для параллельной работы

Разные чаты не должны править `app/dst_0`, `app/dist`, `legacy.css` или `legacy-runtime.js`. Рабочие правки делаются в `app/src` и затем собираются в `app/dist`.
